./tar_projects.sh usage:
executing script without param:
	no tar and no scp is done
typical usage: ./tar_projects.sh -e --scp -v && echo "exit code is $?"
	Will do the tar of tar_source_dir: /home/chris/projects to tar-file projects_20240701.tar.gz
	in tar_dest_dir: /home/chris/backups/ if both the tar_source_dir and the tar_dest_dir exist.

tar command that will be used:
	tar czfv /home/chris/projects /home/chris/backups/projects_20240701.tar.gz

scp command that will be used:
	scp /home/chris/backups/projects_20240701.tar.gz chris@ThinkPad:c:/backups/

Remarks:
	if tar_source_dir and/or tar_dest_dir do not exist the script shows an ERROR and exits.
	tar-file is generated by the script as: projects_YYYYmmdd.tar.gz
	exit-code of the script: 0: all ok; 1: general error; 2: error with tar; 3: error with scp
	scipt has very limited to no error handling; very limited to no testing of the script has been done

Script params:
	-h --help: this help
	-s <dir> --source <dir>: use the <dir> as the source dir for files to tar
	-t <dir> --target <dir>: use the <dir> as the target dir of the tar
	-v --verbose: show more info on script execution
	-y --yes -e --execute: run the tar command, and scp command if tar went fine
	--scp: do scp to windows host
